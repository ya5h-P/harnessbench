littleutils.py is a real utility library (MIT 'littleutils'). Its function group_by_key_func(
iterable, key_func) should build a dict mapping each key (the result of key_func on an element) to
the LIST of all elements with that key, preserving input order, e.g. group_by_key_func('a bb ccc d
ee fff'.split(), len) == {1:['a','d'], 2:['bb','ee'], 3:['ccc','fff']}. It currently has a bug that
keeps only one element per key instead of grouping them into lists. Fix group_by_key_func in
littleutils.py to match its documented behavior. Don't break other functions. Verify by running python.
